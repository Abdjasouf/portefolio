Projet Séminaire

Objectif du projet :

Développement d'une solution complète pour la gestion des inscriptions aux conférences du Centre de congrès de Limoges. Le projet se divise en deux parties : la création d'une API REST sécurisée et d'un site web dynamique consommant cette API.

Outils et Architecture :

  • Backend : PHP 8 (Architecture MVC & POO)
  • API : REST / Format JSON
  • Base de données : MySQL (PDO)
  • Communication : Bibliothèque cURL pour la consommation de l'API

Mes missions :

1. Développement de l'API REST : Conception des points d'entrée (endpoints) pour récupérer les conférences, gérer l'authentification des participants et enregistrer les inscriptions en base de données.

[Image of REST API architecture diagram]

2. Consommation de l'API (Client Web) : Implémentation d'un ApiClient en PHP pour effectuer des requêtes HTTP (GET/POST). Développement du système de session pour maintenir la connexion utilisateur.

3. Interface Utilisateur : Création de vues dynamiques permettant de filtrer les conférences par séminaire et de s'y inscrire en un clic.

Fonctionnalités clés :

  • Authentification sécurisée via l'API
  • Consultation du programme par séminaire
  • Inscription et désinscription aux conférences
  • Consultation de l'historique personnel des inscriptions
