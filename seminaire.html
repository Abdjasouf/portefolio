Projet Séminaire

Contexte :

Projet de gestion d’événements pour un centre de congrès, basé sur une architecture API REST et un site web client.

Objectif du projet :

Permettre aux utilisateurs de consulter des conférences et de s’y inscrire via une application web connectée à une API.

Outils et architecture :

  • Backend : PHP (MVC)
  • API : REST (JSON)
  • Base de données : MySQL (PDO)
  • Communication : cURL

Mes missions :

  • Développement des endpoints de l’API
  • Gestion de l’authentification utilisateur
  • Création d’un client API en PHP
  • Développement des interfaces utilisateur

Compétences acquises :

  • Compréhension des API REST
  • Communication client / serveur
  • Gestion des sessions utilisateurs
  • Développement web structuré
